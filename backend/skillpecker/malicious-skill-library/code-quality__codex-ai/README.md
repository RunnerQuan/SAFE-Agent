# Codex-AI 协作技能

通过 MCP 工具调用 Codex,处理复杂代码审查、算法设计、架构分析和性能优化任务。

## 什么是 Codex-AI?

Codex-AI 是一个智能协作技能,能够:
- 自动识别复杂技术任务并调用 Codex 专家模型
- 根据任务复杂度智能选择合适模型
- 提供结构化的分析报告和可执行建议

## 何时使用?

### 触发关键词

| 场景 | 触发词 | 典型问题 |
|------|--------|---------|
| 代码审查 | review, 审查代码 | "帮我审查未提交的代码" |
| 算法设计 | 算法设计, 复杂逻辑 | "设计一个 LRU 缓存" |
| 架构分析 | 架构评审, 系统设计 | "分析微服务扩展性" |
| 性能优化 | 性能瓶颈, 优化 | "优化 API p99 延迟" |

## 使用示例

### 示例 1: 代码审查

**你说**: "审查一下我未提交的代码变更"

**技能会做什么**:
1. 获取你的 git diff
2. 调用 Codex 进行深度分析
3. 返回审查报告,包括:
   - 潜在问题(空指针/内存泄漏/并发问题)
   - 性能建议
   - 安全检查
   - 代码风格改进

**典型输出**(示意):
```
📊 代码审查结果

发现 3 个问题:

1. 潜在空指针异常
   位置: src/main.rs:45
   建议: 添加 null 检查

2. 性能问题
   位置: src/utils.rs:120
   建议: 使用 HashMap 实现 O(1) 查找

3. 并发安全
   位置: src/cache.rs:88
   建议: 使用 RwLock 保护共享状态
```

## 如何获得最佳效果?

### ✅ 最佳实践

1. **明确约束**: 提供具体性能指标和资源限制
   - ✅ "p99 延迟 500ms,目标 100ms"
   - ❌ "代码很慢"

2. **提供上下文**: 相关代码片段、架构信息、当前瓶颈

3. **具体目标**: 说明期望的输出和格式

### ❌ 避免事项

1. 模糊描述,缺少性能指标
2. 缺少约束条件和边界情况
3. 跳过验证直接实施建议

## 技能如何工作?

1. **任务识别**: 自动识别任务类型(代码审查/算法设计/架构分析/性能优化)
2. **模型选择**: 根据复杂度选择模型
   - 简单任务: gpt-5.2-codex(快速,<10秒)
   - 复杂任务: gpt-5.2(深度推理,20-60秒)
3. **上下文准备**: 收集必要信息(git diff、代码片段等)
4. **调用 Codex**: 通过 MCP 工具发起请求
5. **格式化输出**: 结构化展示分析结果

## 模型选择说明

| 特性 | gpt-5.2-codex | gpt-5.2 |
|------|--------------|---------|
| 速度 | 快(<10s) | 慢(20-60s) |
| 推理深度 | 1-2 层 | 3+ 层 |
| 适用任务 | 代码审查、简单重构 | 架构设计、复杂算法 |
| 约束条件 | 1-2 个 | 3+ 个需权衡 |

## 常见问题

**Q: 技能会自动修改我的代码吗?**

A: 不会。技能只提供分析和建议,不会直接修改代码。

**Q: 如何确保建议是正确的?**

A: 建议在测试环境验证、编写测试用例、逐步实施。

**Q: 审查需要多长时间?**

A: 取决于任务复杂度。简单审查<10秒,复杂分析 20-60秒。

**Q: 可以追问和继续对话吗?**

A: 可以。技能支持多轮对话,你可以追问细节。
